Hexo로 만든 블로그 검색 엔진 최적화(SEO) 하기

2020/08/22 카테고리 blog

목차

  1. SEO에 유용한 플러그인 설치하기
    1. hexo-auto-canonical
    2. hexo-generator-seo-friendly-sitemap

이 글에서는 hexo로 만든 블로그에 검색 엔진 최적화 작업을 해보겠습니다

SEO 란 ?
SEO는 Search Engine Optimization의 약자로, 우리말로는 검색 엔진 최적화라고 합니다 !
구글, 네이버 등의 검색엔진이 자료를 수집하고 순위를 매기는 방식에 맞도록 웹페이지를 구성하는 작업을 의미합니다.
이 작업을 거치면 검색 엔진에서 검색 결과의 상위에 노출될 수 있습니다.
내가 쓴 글이 누군가의 필요에 의해 읽히기를 원하신다면 SEO 작업을 진행해보세요 ٩( ‘ω’ )و

SEO에 유용한 플러그인 설치하기

hexo-auto-canonical

이 플러그인은 블로그의 대표 url을 검색엔진에 알려주는 url 표준화를 담당합니다.

다음 명령어로 플러그인을 먼저 설치해주세요.

npm install hexo-auto-canonical --save

현재 블로그에 적용된 테마 폴더 아래에 layout 폴더에 잘 찾아보시면 head.ejs 파일이 있습니다.

저는 tranquilpeak 테마를 적용하고 있는데 themes/tranquilpeak/layout/_partial/head.ejs 여기에 있네요 !

테마에 따라서 layout/common 폴더에 head.ejs 파일이 있다고도 합니다

찾으신 head.ejs 파일에
<%- autoCanonical(config, page) %>
요 코드를 삽입해 주세요 !

잘 적용되었는지 확인하시려면, hexo server로 로컬 서버를 켠 후에 개발자 도구를 켜주세요 !

세번째 줄에 보이는 것처럼 rel canonical이 생겼다면 성공입니다 (`•ω•´)/*҉

canonical tag는 검색 엔진에게 특정 url이 현재 페이지의 master copy를 대표한다고 알려주는 역할을 한다고 해요 ! 즉, canonical tag를 사용하면 여러개의 url에서 중복된 내용을 보여주는 것을 방지할 수 있다고 합니다.

hexo-generator-seo-friendly-sitemap

이 플러그인은 크롤러가 우리의 블로그를 효율적으로 크롤링 할 수 있도록 사이트맵을 생성해주는 역할을 합니다 !
그러고보면 플러그인 이름이 꽤나 직관적입니다 °o°
요 플러그인이 자동으로 sitemap.xml 을 생성해주어요

다음 명령어로 플러그인을 먼저 설치해주세요

npm install hexo-generator-seo-friendly-sitemap --save

hexo-generator-robotstxt

이 플러그인은 자동으로 robot.txt 파일을 생성해줍니다. 우리말로 로봇 배제 표준 이라고 하는 이 문서는, 웹 사이트에 로봇이 접근하는 것을 방지하기 위한 규약을 기술한 문서입니다.

다음 명령어로 플러그인을 설치해봅시다

npm install hexo-generator-robotstxt --save
