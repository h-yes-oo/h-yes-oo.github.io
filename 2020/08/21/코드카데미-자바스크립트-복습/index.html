Functions as Parameters

.forEach()

  • array method 중 하나 ArrayName.forEach() 로 호출
  • .forEach() 는 argument 로 callback function을 가짐

    callback function : 다른 함수의 argument로 passed되는 함수

  • .forEach() 는 array의 각 요소를 callback function의 argument로 받아 실행
  • .forEach()의 return 값은 항상 undefined

.map()

  • array method 중 하나 ArrayName.map() 로 호출
  • argument 로 callback function을 가짐
  • array의 각 요소를 callback function의 argument로 받아 실행하고, callback function의 return 값을 새로운 array에 저장
  • return 값은 새롭게 만들어진 array

.filter()

  • callback function이 array의 각 요소를 argument로 받고, true 혹은 false를 반환
  • true를 반환하는 요소만 담은 새로운 array를 return함

.findIndex()

  • callback function이 true를 반환하게 하는 첫 요소의 인덱스를 return함

.reduce()

  • arr.reduce(callback( accumulator, currentValue[, index[, array]] )[, initialValue]) reduce의 parameter로 initialValue는 생략가능
  • accumulator : 이전 콜백 함수에서 return된 값 혹은 reduce의 parameter로 주어진 initialValue
  • currentValue : arr의 원소가 차례로 들어감

.some() .every()

  • callback ftn의 return값이 true인 원소가 있는지, 혹은 모든 원소가 true값을 반환하는지 boolean으로 return

npm 이란 ?

초기의 자바스크립트 프로젝트 파일 구조

여기에 npm init 명령을 실행시키면, package.json 파일이 root directory에 생성됨

package.json 파일은 현재 자바스크립트 프로젝트에 관한 정보들을 담고 있는데,

  • Metadata (project title, description, authors, … )
  • 프로젝트에 필요한 노드 패키지들의 목록
  • Key-value pairs for command line scripts

npm이 package.json 파일을 src 폴더와 같은 위치에 생성함

뒤에 -D 옵션을 붙이면, package.json의 devDependencies 속성에 필요한 환경 패키지들을 모두 등록함

Modules

JavaScript 모듈이란, 한 프로그램에서 다른 프로그램으로 export 될 수 있는 재사용 가능한 코드의 덩어리를 의미한다.

코드를 모듈로 쪼갬으로써,

  • 코드를 쉽게 찾고, 고치고, 디버깅할 수 있다
  • 우리 어플리케이션의 서로 다른 부분에서 로직을 재활용할 수 있다
  • 정보를 다른 모듈로부터 private 하도록 지킬 수 있다
  • potential namaing collision을 방지할 수 있다

Promises

Promise 객체는 asynchronous operation의 최종 결과물을 대표하는 객체이다.

The Node setTimeout() Function

setTimeout() 은 Node API로, callback function을 이용하여 delay 이후에 task가 수행되도록 스케쥴한다.

setTimeout() 은 callback function과 delay in milliseconds의 두개의 파라미터 갖는다.

동기(synchronous)와 비동기(Asynchronous)
동기란 말 그대로 동시에 일어난다는 뜻으로, 요청과 그 결과가 동시에 일어난다는 약속이다. 요청한 자리에서 바로 결과가 주어진다.
결과가 주어질 때까지 아무것도 못하고 대기해야 하는 단점이 있다.


비동기란 동시에 일어나지 않는다는 것을 뜻한다. 요청과 결과가 동시에 일어나진 않을 것이라는 약속이다.
요청한 그 자리에서 결과가 주어지진 않는다.
결과가 주어지는데에 시간이 걸리더라도 그 시간 동안 다른 작업을 할 수 있다.
