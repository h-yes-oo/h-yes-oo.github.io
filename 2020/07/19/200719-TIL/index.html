개발 공부 원칙

  1. 선 실습 + 후 이론
  2. 다양한 활동에 참여하기 - 컨트리뷰톤, 자기보다 잘하는 사람과 외주하기
  3. Today I Learned - 잘 모르고 지나가는 개념들 정리해서 올리기(예 Runtime, …)
  4. 1일 1커밋
  5. 팀 / 개인 프로젝트 - 완성을 목표로 하되, 끝까지 하지 않아도 의미가 있음 (제대로 된 것 2개면 충분, 시의성이 있다면 플러스)
  6. 한계효용 체감의 법칙 - 짧은 시간에 빡세게 하는 것이 효율적
  7. 멋사 사람들 잘 활용하기 - 질문 많이 하기

서류 합격에 도움이 되는 것

  1. 학교
  2. Github - 잔디 잘 심기, 제대로 만든 프로젝트 pin 걸기, 오픈소스 뱃지 달기, 개발 할 때 기록 꼭 잘 남겨놓기(커밋 메세지, 코드 리뷰 등 제대로 할 것)
  3. 포트폴리오 예) 현상 오빠 포트폴리오
  4. 개발 블로그(TIL)

면접 합격에 도움이 되는 것 ( 가능성을 보여주기 )

  1. 개발 블로그
  2. 한 언어 깊게 파기
  3. 써본 기술들 + 경쟁 기술들 파악 (예> 왜 장고로 서비스를 만들었는지, db 선택한 이유 무엇인지) 본인이 쓰는 기술을 왜 쓰는지 알아둘 것
  4. 찝찝한 거 내버려두지 말기, 허를 찔릴 수 있음
  5. 가고 싶은 회사에서 쓰는 기술 습득
  6. T자 형태의 공부 - 면접에 실제로 나오는 범위는 생각보다 좁음
  7. 경험과 지식을 미리 엮어두기
  8. 이론은 토픽단위로 공부하기 - 검색어 단위로

총평

  1. 공부해야할 것들이 생각보다 한정되어 있음
  2. 현상 오빠가 했던 것들 - 멋사, 토이 프로젝트, 포트폴리오, 개발 블로그, 컨트리뷰톤
  3. 생각보다 안 중요했던 것 - 배포, 어려운 알고리즘, 리액트 스킬(코드레벨 지식), 객체지향/함수형 등, 로우레벨 언어(c언어 몰라도 돼요), git
  4. 생각보다 중요했던 것들 - 쉬운 자료구조(특징들), 쉬운 알고리즘(정렬, 시간복잡도)

프론트 개발자가 준비해야할 것들

필수 > 자바스크립트, 기본 자료구조 알고리즘 >> 리액트
선택 > 웹팩/바벨, SQL(ORM때문에 알기 어려움), 타입스크립트(자스 잘하면 하루만에도 가능), 노드(express or Koa), 몽고DB
이미 배운거라고 가정> 제이쿼리, sqlite

  • 대기업 채용을 목표로 하더라도 신입 채용보다 경력 채용이 훨씬 쉬우므로 처음부터 대기업에 갈 필요는 없다. 단 첫 직장 연봉이 중요하므로 잘 생각할 것
